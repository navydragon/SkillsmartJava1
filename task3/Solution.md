task3 — Три кита ООП
================================================================

3.1. Инкапсуляция
-----------------

**Реализация:**
- Все поля классов (`Spaceship`, `Pilot`, `Mission`, `Vehicle`) сделаны `private`
- Доступ к полям организован через геттеры (методы `get...()`) и сеттеры (методы `set...()`)
- Сеттеры контролируют корректность значений (например, `stressLevel` ограничен 0.0-1.0, `riskLevel` — 1-10)

**Примеры:**
- `Pilot.setStressLevel()` — не допускает отрицательных значений и значений > 1.0
- `Mission.setRiskLevel()` — ограничивает риск в диапазоне 1-10
- `Vehicle.setRemainingFuel()` — ограничивает топливо в диапазоне 0-100%

3.2. Наследование
-----------------

**Иерархия классов:**
```
Vehicle (базовый класс)
  └── Spaceship (наследуется от Vehicle)
      ├── ExplorerShip (наследуется от Spaceship)
      └── CourierShip (наследуется от Spaceship)
```


3.3. Переопределение методов


**Примеры переопределения:**

1. **Vehicle.move()** — базовый метод:
   - Расход топлива: `distance * 0.1`

2. **Spaceship.move()** — переопределён для кораблей:
   - Расход топлива: `distance * 0.15` (больше из-за размера)
   - Выводит сообщение о полёте

3. **ExplorerShip.move()** — переопределён для исследовательских кораблей:
   - Расход топлива: `distance * 0.2` (ещё больше из-за оборудования)
   - Выводит информацию об исследовательских модулях

4. **CourierShip.move()** — переопределён для курьерских кораблей:
   - Расход топлива: `distance * 0.08` (меньше, более экономичные)
   - Автоматически ускоряется при движении
   - Выводит информацию о грузе

**Специфичные методы:**
- `ExplorerShip.conductResearch()` — метод только для исследовательских кораблей
- `CourierShip.deliverCargo()` — метод только для курьерских кораблей



3.3. Демонстрация работы
-------------------------
В `Main.java` показаны примеры:
- Инкапсуляции: использование геттеров/сеттеров, контроль граничных значений
- Наследования: создание объектов дочерних классов, использование унаследованных методов
- Переопределения: вызов переопределённых методов, демонстрация разного поведения
