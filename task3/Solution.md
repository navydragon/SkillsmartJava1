task3 — Три кита ООП
================================================================

3.1. Инкапсуляция
-----------------
Инкапсуляция — разделение содержимого класса на видимые извне и невидимые извне.

**Модификаторы доступа:**
- `public` — виден без ограничений
- `private` — виден только внутри данного класса
- `protected` — виден только внутри данного класса и его потомков

**Реализация:**
- Все поля классов (`Spaceship`, `Pilot`, `Mission`, `Vehicle`) сделаны `private`
- Доступ к полям организован через геттеры (методы `get...()`) и сеттеры (методы `set...()`)
- Сеттеры контролируют корректность значений (например, `stressLevel` ограничен 0.0-1.0, `riskLevel` — 1-10)

**Примеры:**
- `Pilot.setStressLevel()` — не допускает отрицательных значений и значений > 1.0
- `Mission.setRiskLevel()` — ограничивает риск в диапазоне 1-10
- `Vehicle.setRemainingFuel()` — ограничивает топливо в диапазоне 0-100%

3.2. Наследование
-----------------
Наследование позволяет создавать иерархию классов, где дочерние классы наследуют свойства и методы родительских.

**Иерархия классов:**
```
Vehicle (базовый класс)
  └── Spaceship (наследуется от Vehicle)
      ├── ExplorerShip (наследуется от Spaceship)
      └── CourierShip (наследуется от Spaceship)
```

**Ключевые моменты:**
- Использование ключевого слова `extends` для наследования
- Вызов конструктора родительского класса через `super()`
- Дочерние классы имеют доступ к `protected` полям и методам родителя
- Дочерние классы могут добавлять собственные поля и методы

**Примеры:**
- `Spaceship extends Vehicle` — корабль наследует общие свойства транспортного средства
- `ExplorerShip extends Spaceship` — исследовательский корабль наследует свойства корабля
- `CourierShip extends Spaceship` — курьерский корабль наследует свойства корабля

3.3. Переопределение методов
----------------------------
Переопределение методов позволяет дочерним классам изменять поведение методов, унаследованных от родительского класса.

**Реализация:**
- Использование аннотации `@Override` для явного указания переопределения
- Имя и параметры переопределяемого метода должны точно соответствовать родительским
- При вызове метода на объекте дочернего класса выполняется переопределённая версия

**Примеры переопределения:**

1. **Vehicle.move()** — базовый метод:
   - Расход топлива: `distance * 0.1`

2. **Spaceship.move()** — переопределён для кораблей:
   - Расход топлива: `distance * 0.15` (больше из-за размера)
   - Выводит сообщение о полёте

3. **ExplorerShip.move()** — переопределён для исследовательских кораблей:
   - Расход топлива: `distance * 0.2` (ещё больше из-за оборудования)
   - Выводит информацию об исследовательских модулях

4. **CourierShip.move()** — переопределён для курьерских кораблей:
   - Расход топлива: `distance * 0.08` (меньше, более экономичные)
   - Автоматически ускоряется при движении
   - Выводит информацию о грузе

**Специфичные методы:**
- `ExplorerShip.conductResearch()` — метод только для исследовательских кораблей
- `CourierShip.deliverCargo()` — метод только для курьерских кораблей

3.4. Принципы реализации
-----------------------
- **Инкапсуляция:** Все поля приватные, доступ через геттеры/сеттеры с контролем значений
- **Наследование:** Иерархия классов с использованием `extends` и `super()`
- **Переопределение:** Методы переопределяются в дочерних классах для изменения поведения
- **Контроль состояния:** Сеттеры гарантируют, что объекты не попадут в некорректное состояние

3.5. Демонстрация работы
-------------------------
В `Main.java` показаны примеры:
- Инкапсуляции: использование геттеров/сеттеров, контроль граничных значений
- Наследования: создание объектов дочерних классов, использование унаследованных методов
- Переопределения: вызов переопределённых методов, демонстрация разного поведения
